# VOIDNET RFC Index

Status: draft

VOIDNET uses RFCs to freeze protocol decisions before they become implementation commitments. RFCs are intentionally narrow. They describe objective, scope, assumptions, and future considerations without hiding unresolved questions.

The existing [RFC 0001: VOIDNET Network Foundation](0001-network-foundation.md) remains as the original foundation memo. The canonical protocol RFC series begins below.

## Canonical RFC Series

- [RFC-0001 VOID Protocol](0001-void-protocol.md)
- [RFC-0002 VOIDDNS](0002-voiddns.md)
- [RFC-0003 VOID Identity](0003-void-identity.md)

- [RFC-0005 Distributed State](0005-distributed-state.md)
- [RFC-0006 Transport Layer](0006-transport-layer.md)
- [RFC-0007 Event Bus](0007-event-bus.md)

## RFC Template

Each RFC should define:

- Status.
- Objective.
- Scope.
- Protocol assumptions.
- Future considerations.

Optional sections may include wire format, security model, failure modes, compatibility, and implementation notes.
